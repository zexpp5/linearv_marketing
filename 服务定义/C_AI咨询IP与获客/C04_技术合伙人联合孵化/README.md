# 方向 11：技术合伙人 / 联合孵化

> 不赚现金流，赚期权——用技术交付换股权/收入分成。

> **状态：后置，依赖方向 2 的咨询 IP 先建立起来**

## 产品形态

- 按股权/收入分成换 AI 技术交付
- AI MVP 两周出 demo 的快交付
- 创始人 AI 技术顾问（按月）

## 前置条件：咨询 IP

技术合伙人模式的核心不是技术，是**信任**。需要先通过方向 2（咨询培训）建立 IP：

```
方向 2 咨询培训 → 建立行业认知和 IP
    ↓
通过咨询接触大量早期项目
    ↓
筛选 1-2 个高潜项目
    ↓
以技术合伙人身份介入
```

### 筛选标准（什么项目值得下注）

- 创始人有行业资源和销售能力（不只是技术想法）
- AI 是核心竞争力而非锦上添花
- 市场规模足够大，且有明确的付费意愿验证
- 我们的技术栈能覆盖其核心需求

### 时间线建议

1. **现在 → 6 个月**：专注方向 2 建立咨询 IP，积累项目接触量
2. **6 个月后**：从咨询客户中筛选候选项目
3. **每年最多投入 1-2 个**：控制风险敞口

## 商业模型

| 维度 | 说明 |
|------|------|
| 客户买单逻辑 | 早期团队没钱但有方向，用股权换技术合伙人 |
| 典型客单价 | 股权 5-20% / 收入分成 10-30% |
| 收费模式 | 股权 / Revenue Share / 象征性月费 |
| 交付难度 | 高 |
| 现金流速度 | 几乎为零（期权属性） |
| 战略价值 | 挑 1-2 个下注，一个命中顶前面所有业务 |

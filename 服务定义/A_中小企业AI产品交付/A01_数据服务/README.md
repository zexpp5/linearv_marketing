# 方向 8：数据服务（被严重低估的底座）

> 数据是 AI 项目做不好的 80% 原因，必须单独报价。

## 产品形态

- 企业文档清洗、标注、结构化
- 知识库从 0 到 1 的一次性搭建
- 非结构化数据资产化（合同/客诉/工单挖掘）
- 私有数据脱敏与向量化

## 传统数据服务 vs 我们的 AI 化做法

| 环节 | 传统做法 | AI 化做法 |
|------|----------|----------|
| 文档清洗 | 人工逐条整理 | AI 自动提取结构 + 人工抽检 |
| 数据标注 | 标注团队人肉标 | AI 预标注 + 人工校正（效率提升 5-10x） |
| 知识库搭建 | 人工分类整理 → 传统搜索 | AI 自动分块 + 向量化 → 语义检索 |
| 数据脱敏 | 正则规则 + 人工审核 | NER 模型自动识别敏感信息 + 规则兜底 |
| 数据资产化 | BI 报表 | AI 自动生成洞察 + 自然语言查询 |

## 核心差异化：用 AI 做数据服务

**关键卖点：同样的活，我们用 AI 做更快更便宜**

1. **自动化程度高** — 80% 工作 AI 完成，人工只做校验和兜底
2. **交付带产品** — 不只交数据，还交一个可持续使用的数据处理工具/流程
3. **与后续 AI 项目无缝衔接** — 数据处理完直接灌入方向 1 的场景包或方向 4 的知识库产品

## 产品化方向

### 数据治理工具箱（可复用）
- 文档自动分块与结构化提取工具
- AI 预标注 + 人工校正平台
- 敏感信息自动识别与脱敏工具
- 数据质量自动评估报告

### 服务交付流程
```
客户原始数据接入
    ↓
AI 自动诊断（数据质量报告）
    ↓
自动清洗 + 结构化
    ↓
人工抽检校正
    ↓
向量化 / 入库
    ↓
交付数据资产 + 处理工具 + 质量报告
```

## 商业模型

| 维度 | 说明 |
|------|------|
| 客户买单逻辑 | 想做 AI 但数据一团乱，必须先处理 |
| 典型客单价 | 5-30 万 / 项目 |
| 收费模式 | 项目制 + 后续迭代 |
| 交付难度 | 中高（人力密集）→ AI 化后降为中 |
| 现金流速度 | 中 |
| 战略价值 | 所有 RAG/Agent 项目的前置必做环节，单独卖利润更好 |

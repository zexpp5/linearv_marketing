# 方向 4：托管运营服务（MSP 模式）

> 卖完不是结束，是开始；按月收费帮客户运营 AI。

## 产品形态

原始服务形态：
- 知识库持续更新与调优
- Prompt 迭代优化
- 模型效果监控与调整
- 用量优化（帮省 token 费用）

## 需要产品化的方向

这些服务不能只靠人力堆，必须做成产品才能规模化。

### 产品 1：企业知识库产品

| 项目 | 说明 |
|------|------|
| 核心功能 | 文档上传 → 自动分块 → 向量化 → 智能检索 → 问答 |
| 差异化 | 支持多源接入（飞书/钉钉/语雀/本地文件）、权限管理、版本追踪 |
| 付费模式 | SaaS 月费（按文档量/用户数计费） |
| 目标客户 | 已有大量内部文档但检索效率低的中小企业 |
| 关键指标 | 检索准确率、首次回答解决率、知识更新延迟 |

### 产品 2：LLM 网关与费用审计

| 项目 | 说明 |
|------|------|
| 核心功能 | 统一 API 网关（对接多模型）、用量监控、费用分账、预算告警 |
| 差异化 | 帮企业管住 token 开支，按部门/项目/场景拆账 |
| 付费模式 | 基础免费 + 按调用量阶梯计费 |
| 目标客户 | 已在用多个 AI 模型但费用不透明的团队 |
| 关键指标 | 月度 token 消耗、费用节省比例、模型切换成功率 |

### 产品 3：Agent 编排管理平台

| 项目 | 说明 |
|------|------|
| 核心功能 | Prompt 版本管理、Agent 工作流编排、效果监控与 A/B 测试、自动回滚 |
| 差异化 | 非技术人员也能调整 Prompt 和工作流，效果有数据可追踪 |
| 付费模式 | SaaS 月费（按 Agent 数量/调用量计费） |
| 目标客户 | 已部署 AI 场景但缺运维能力的企业 |
| 关键指标 | Agent 成功率、Prompt 迭代频率、异常响应时间 |

## 产品与服务的关系

```
客户购买场景包(方向1) 或 定制开发(方向3)
         ↓
   交付上线后
         ↓
  托管运营服务(人工) + 产品工具(自动化)
         ↓
  月费订阅（产品费 + 运维服务费）
```

## 商业模型

| 维度 | 说明 |
|------|------|
| 客户买单逻辑 | 企业内部没有 AI 工程师，交出去最省心 |
| 典型客单价 | 3000-20000 元/月 |
| 收费模式 | 月费/年费订阅 |
| 交付难度 | 中（持续投入） |
| 现金流速度 | 慢启动，但续费率最高 |
| 战略价值 | 客户深度绑定的关键，所有其他业务的 LTV 放大器 |

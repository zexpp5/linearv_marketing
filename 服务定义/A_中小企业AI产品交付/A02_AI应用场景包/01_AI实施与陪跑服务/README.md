## 真实失败场景（搜索验证）

### 场景 1：Dify / 知识库 — 部署到一半弃坑

企业 IT 人员部署 Dify，卡在 Docker 环境、端口冲突、Windows 兼容。知识库上传文档后"答非所问"— 根因是文档未做分块预处理，非结构化 PDF/Word 直接丢进去，向量化质量极差。

**我们做什么**：帮部署 + 文档预处理 + 分块策略调优 + 测试验收

### 场景 2：RPA 流程机器人 — 上线即废

中小制造商用 RPA 自动化报销/订单录入，第三方 ERP 一次升级，界面元素变动，所有脚本失效。"维护 RPA 比维护人更贵"。

**我们做什么**：评估是否适合 RPA，推荐 AI 替代方案，或帮维护已有 RPA

### 场景 3：AI 选品工具 — 功能强但没人用

零售商采购 AI 智能选品系统，界面过于专业，门店经理不会操作。工具设计者不懂业务，业务人员不懂工具。

**我们做什么**：做"需求翻译官"，把业务流程和工具逻辑对齐 + 员工培训

### 场景 4：照搬大厂方案 — 规模不匹配

生鲜电商复制头部平台推荐算法，忽视自身用户少、SKU 有限，上线后销量未涨反降。

**我们做什么**：先做场景筛选，帮客户判断哪些 AI 值得做、哪些是伪需求

### 场景 5：用 AI 做 Excel 能干的事

企业用大模型做日常销售报表分析，学习成本+使用成本+校正成本，远超直接用 BI。

**我们做什么**：ROI 评估，帮客户避免"为做 AI 而做 AI"

### 场景 6：数字化地基为零就上 AI

小工厂数据散在纸质记录、手机截图、不同格式 Excel 里，AI 无从训练。

**我们做什么**：先做数据资产整理（联动 A01 数据服务），再上 AI

## 失败的共性规律

| 模式 | 本质原因 |
|------|----------|
| 技术部门孤立推进 | 业务不参与需求定义，上线后"看不懂也用不上" |
| 跨部门协作断裂 | 数据共享、利益分配未提前对齐 |
| 期望管理失控 | 老板期待"一夜颠覆"，员工抵触变革 |
| 照搬成功案例 | 忽视自身规模、数据量、人员能力差异 |
| 无人维护 | 部署完就结束，缺持续迭代和内部 owner |

## 服务产品化

### 轻量包：AI 工具激活（1-3 天）

| 项目 | 说明 |
|------|------|
| 内容 | 诊断已有工具使用情况 → 选 1 个最痛场景 → 配好 AI 工作流 |
| 产出 | 可用的 AI 工作流 + 《AI 工具使用 SOP》 |
| 报价 | **5,000 - 15,000 元** |
| 适合 | 已买 Dify/Coze/飞书 AI 但用不起来的团队 |

### 标准包：AI 场景落地（1-2 周）

| 项目 | 说明 |
|------|------|
| 内容 | 搭建 1-2 个完整场景（知识库/客服/工作流）+ 知识库从 0 搭建 |
| 产出 | 上线运行的 AI 场景 + 员工培训 + 使用 SOP |
| 报价 | **2-5 万** |
| 适合 | 买了智齿/企微客服但知识库空的、有 CRM 但 AI 功能没配的 |

### 深度包：AI 全面落地（1-2 月）

| 项目 | 说明 |
|------|------|
| 内容 | 梳理 3-5 个核心场景 → 逐个落地 → 每周复盘 |
| 产出 | 多个运行中的 AI 场景 + 效果数据 + 迭代计划 |
| 报价 | **5-15 万** |
| 适合 | 想全面 AI 化但不知道从哪开始的企业 |

## 客户需要的五种服务能力

1. **需求翻译官** — 把业务问题翻译成可落地的 AI 任务
2. **数字化地基** — 在上 AI 之前先帮整理数据资产
3. **陪跑式实施** — 部署不是终点，要做培训、推广、迭代
4. **场景筛选顾问** — 帮判断哪些场景值得做、哪些是伪需求
5. **维护兜底** — 上线后的日常维护和系统变更响应

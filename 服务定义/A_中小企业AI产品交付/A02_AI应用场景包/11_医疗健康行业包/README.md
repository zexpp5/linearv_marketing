# 医疗/健康行业场景包

> AI 辅助随访、病历管理、文献检索，释放医护人员时间。

## 业务痛点

| 痛点 | 具体表现 |
|------|----------|
| 随访人力不足 | 慢病患者需定期随访，一个护士管 200+ 患者，电话打不完 |
| 病历非结构化 | 大量手写/语音病历无法检索和分析，科研数据提取靠人工 |
| 文献检索效率低 | 医生查一个问题要翻多篇论文，英文文献更耗时 |
| 基层诊疗能力弱 | 基层医院缺专科医生，诊断质量不均衡 |
| 报告解读滞后 | 影像/检验报告人工解读等待时间长 |

## 场景清单

| 场景 | 说明 |
|------|------|
| 患者随访助手 | 自动外呼 + 记录 + 异常标记 |
| 病历结构化与检索 | 非结构化病历自动提取关键信息，支持语义检索 |
| 医学文献摘要与问答 | 上传论文/指南，AI 自动摘要并回答问题 |

## 目标客户

- 民营医院/诊所
- 健康管理公司
- 医药企业（医学信息部）

### 市场数据

- 2023 年中国 AI 医疗市场 88 亿元，预计 2033 年达 3,157 亿元（CAGR 43.1%）
- 商业模式以 B 端医院 SaaS + 项目制为主，公开定价极少

### 我们的差异化机会

- **竞品问题**：头部产品（讯飞/森亿/医渡）面向三甲和大型医院，项目制动辄百万；影像 AI 门槛极高（需 NMPA 认证）
- **我们的切入点**：
  - 不做影像 AI 和辅助诊断（合规门槛太高）
  - 聚焦**非临床场景**：随访自动化、病历结构化检索、文献问答
  - 面向**民营医院/诊所/健康管理公司**，这些客户被头部厂商忽略
  - 复用底层 RAG + 文档处理能力，本质是"给医疗行业做知识问答"

## 注意事项

- 医疗场景合规要求高，需明确 AI 辅助定位（不做诊断）
- 数据隐私要求严格，需脱敏处理
- 涉及医疗数据需关注《数据安全法》《个人信息保护法》合规

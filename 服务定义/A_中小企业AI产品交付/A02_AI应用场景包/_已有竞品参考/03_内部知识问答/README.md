# 内部知识问答

> 接飞书/钉钉/语雀，员工问什么都能秒答，告别"翻文档找半天"。

## 业务痛点

| 痛点 | 具体表现 |
|------|----------|
| 找文档耗时 | 员工平均每天花 30-60 分钟找内部资料，新人更久 |
| 知识随人走 | 核心员工离职，经验和流程知识直接断档 |
| 重复提问 | HR/行政/IT 每天回答大量重复问题（报销流程、请假规则等） |
| 文档散落 | 飞书、语雀、本地文件、微信群各存一份，版本不一致 |
| 新人上手慢 | 入职培训 1-2 周，真正独立工作需要 1-2 个月 |
| 制度更新不同步 | 制度改了但没人通知，员工按旧流程操作出错 |

## 核心功能

- 对接企业文档系统（飞书文档、语雀、Confluence、本地文件）
- 自然语言提问 → 精准回答 + 引用来源
- 权限管理（按部门/角色控制可访问知识范围）
- 知识库自动更新（文档变更自动同步）
- 高频问题统计与知识缺口分析

## 目标客户

- 30 人以上团队（内部文档开始难找）
- 新人入职频繁的快速扩张公司
- 有大量 SOP/制度文档的企业

### 我们的差异化机会

- **竞品问题**：
  - 开源产品（FastGPT/Dify/MaxKB）功能强但需要技术团队部署和维护
  - 平台原生（飞书/钉钉）绑定特定生态，跨平台能力弱
  - 两者都不解决"知识库从 0 到 1 搭建"的问题
- **我们的切入点**：
  - **交钥匙服务**：帮客户从文档整理到知识库上线全包
  - 联动 A01（数据服务）做知识库搭建，联动 A05（托管运营）做持续更新
  - 不绑定平台，同时支持飞书/钉钉/企微/Web

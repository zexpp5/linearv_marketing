# 03 跨系统 AI 集成（胶水层）

> 客户已有 3-5 个系统，但数据不通、流程断裂 — 我们用 AI 把它们串起来。

## 市场机会

中小企业真实 IT 现状：飞书/钉钉 + 某 CRM + 企微 + Excel/进销存 + 财务软件，**每个系统都有数据，但互不相通**。

**大厂为什么不做**：每家只管自己的系统，没人有动力帮你打通竞对的产品。

## 真实断点场景（搜索验证）

| 断点 | 具体表现 |
|------|----------|
| 销售 → 财务 | CRM 录入订单后，手动导出 Excel 再导入用友/金蝶，漏单、账期混乱 |
| OA 审批 → ERP | 钉钉/飞书采购审批通过，ERP 采购单仍需人工创建，交货延误 |
| 电商 → 仓储 | 天猫/抖音订单无法自动同步 WMS，旺季人工转单成瓶颈 |
| 客服 → CRM | 企微客服聊天记录与 CRM 割裂，客户画像不完整 |
| 门店 POS → 总部 | 连锁门店 POS 数据各自独立，总部依赖每日手工报表 |

## 现有 iPaaS 工具的根本局限

| 问题 | 说明 |
|------|------|
| 只管搬运，不懂业务 | 字段映射靠人工配，"客户名称" vs "甲方全称"需一一对应 |
| 无法处理异常 | 数据不匹配时只能报错，无法判断改哪边 |
| 国内应用覆盖不足 | 管家婆、畅捷通等本土 ERP 接口不标准，连接器缺失 |
| 非技术人员用不了 | "无代码"实际仍需配置流程逻辑、调试触发条件 |
| 维护成本高 | 业务变更后需重新配置，中小企业无专职 IT |

### 竞品对比

| 产品 | 定位 | AI 能力 | 局限 |
|------|------|---------|------|
| **集简云** | 国内 iPaaS | 2025 年加 AI 节点（接 DeepSeek） | 本质仍是传统规则引擎加 AI 插件 |
| **数环通** | 国内 iPaaS | 类似集简云 | 非原生 AI 架构 |
| **n8n** | 开源工作流 | 支持 LangChain/AI 节点 | 需要技术能力，中小企业用不了 |
| **简道云** | 低代码数据中台 | 有 AI 化苗头 | 偏表单/数据中台，不做系统集成 |
| **蓝凌 MK** | OA + 1000 款软件集成 | 有 | 面向中大型企业 |

### 真空地带

**面向非技术中小企业、原生 AI 驱动、聚焦国内本土系统组合（管家婆+企微+抖音小店等）的轻量集成产品 — 市场空白。**

## AI 在胶水层的独特价值（超越简单数据管道）

| 能力 | iPaaS 做不到，AI 能做 |
|------|---------------------|
| 语义字段映射 | 自动理解"联系人"="客户经理"="负责人"是同一概念 |
| 异常判断与自修复 | 数据冲突时 AI 决策：新客户建档 or 老客户合并 |
| 非结构化数据提取 | 从企微聊天、邮件、PDF 报价单中提取关键字段写入系统 |
| 自然语言配置 | 业务人员用中文描述需求，AI 生成自动化规则 |
| 主动预警 | 跨系统数据比对发现异常（如库存不一致时告警） |

## 典型集成场景

### 场景 1：企微/微信 → CRM 自动同步

| 维度 | 说明 |
|------|------|
| 做什么 | 企微聊天记录 AI 摘要 → 写入 CRM 跟进记录 → 自动打标签 |
| 技术 | 企微 API + LLM 摘要 + CRM API |
| 报价 | 3-6 万 |

### 场景 2：多平台订单 → 统一看板 + AI 日报

| 维度 | 说明 |
|------|------|
| 做什么 | 淘宝/京东/抖音/独立站订单汇总 → AI 自动生成经营日报 |
| 技术 | 各平台 API + 数据聚合 + LLM 报告生成 |
| 报价 | 3-8 万 |

### 场景 3：审批 → 财务自动记账

| 维度 | 说明 |
|------|------|
| 做什么 | 报销审批通过 → AI 识别发票 → 生成凭证 → 推送财务系统 |
| 技术 | 飞书/钉钉 Webhook + OCR + 财务 API |
| 报价 | 2-5 万 |

### 场景 4：客户全渠道画像

| 维度 | 说明 |
|------|------|
| 做什么 | 微信/电话/邮件所有沟通 → AI 汇总客户画像 → 同步 CRM/工单 |
| 技术 | 多渠道采集 + LLM 摘要分类 + 统一视图 |
| 报价 | 5-10 万 |

### 场景 5：门店巡检 → 自动整改报告

| 维度 | 说明 |
|------|------|
| 做什么 | 拍照+语音备注 → AI 生成整改报告 → 推送负责人 |
| 技术 | 小程序 + 多模态 LLM + 通知 API |
| 报价 | 2-5 万 |

## 技术栈复用

所有场景底层能力同一套：
- API 对接（各系统 API/Webhook）
- LLM 调用（摘要/分类/生成）
- 工作流引擎（n8n/Dify/自建）

**做一个场景积累的能力，直接复用到下一个。**

## 竞争优势

| 优势 | 说明 |
|------|------|
| 没有直接竞品 | iPaaS 只做数据搬运，不做 AI 理解和生成 |
| 交付快 | 1-3 周，不是传统集成 3-6 个月 |
| 客单价合理 | 2-10 万，中小企业出得起 |
| 粘性强 | 一旦用起来就离不开，天然导入 A05 托管运营 |

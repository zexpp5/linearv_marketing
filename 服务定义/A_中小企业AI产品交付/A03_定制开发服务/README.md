# 方向 3：定制开发服务（AI 外包开发）

> 接住那些没法标品化的长尾需求，按项目报价。本质是 AI 时代的外包开发。

## 产品形态

- 小程序/网页版 AI 工具开发
- 已有系统的 AI 能力改造（给老 CRM/ERP 加 AI）
- 企业微信/飞书/钉钉机器人开发

## 与传统外包的差异化

| 维度 | 传统外包 | 我们的 AI 外包 |
|------|----------|---------------|
| 交付物 | 功能模块 | 功能模块 + AI 能力 + Prompt/模型调优 |
| 壁垒 | 低，拼人头价格 | 中，需要 AI 工程经验 |
| 复用性 | 低 | 高（底层 RAG/Agent 架构可复用） |
| 后续粘性 | 弱 | 强（AI 需要持续调优 → 导入方向 4 托管运营） |

## 定位策略

- **不主推，但不能没有** — 作为方向 1（场景包）覆盖不到的长尾补充
- 每个定制项目尽量抽象出可复用模块，反哺标准场景包
- 优先接与现有技术栈匹配度高的项目，避免纯堆人头

## 商业模型

| 维度 | 说明 |
|------|------|
| 客户买单逻辑 | 现有系统加 AI、老板想做一个"自己的 AI 工具" |
| 典型客单价 | 5-30 万 / 项目 |
| 收费模式 | 项目制一次性收费 + 后续维护 |
| 交付难度 | 高（需求多变、易扯皮） |
| 现金流速度 | 慢（周期 1-3 个月） |
| 战略价值 | 长尾收入来源，非主力但不能没有 |

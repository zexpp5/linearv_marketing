# 公司定位研究项目

本目录用于沉淀 LinearVision 的公司定位基础资料。当前已经基于 2026-04-10 的讨论形成一版基础定位，后续文档先围绕这版统一收敛。

## 当前版本

- 项目版本：`v0.4.2`
- 版本说明：新增广告目录与潜在广告语文档

## 版本规则

- `major`：定位方向、核心结构或方法论发生明显重构
- `minor`：新增重要模块、完整章节或关键结论
- `patch`：措辞优化、局部补充、格式整理

## 当前已确认的基础定位

- 品牌主张：`LinearVision — See. Think. Act. With AI.`
- 中文辅助表达：`看见、思考、行动，AI 与你并肩`
- 核心逻辑：围绕 `See / Think / Act` 三个阶段组织品牌和产品表达
- 目标客户：银行与金融机构、投资机构、科技公司、创业团队
- 公司类型：基于 Claude、GPT 等顶尖大模型，为企业构建 AI 决策与执行系统的应用方案商，而不是基础模型研发公司
- 当前明确产品：`AI Agent绩效`、`AI安全`

## 目录结构

```text
公司定位/
├── 01_现状盘点
├── 02_外部研究
├── 03_定位与品牌
└── 04_附件资料
```

## 目录说明

- `01_现状盘点`：已确认的公司信息、表达边界、当前阶段判断
- `02_外部研究`：四类目标客群的差异化信息需求、市场阶段判断、行业观察与竞对分析
- `03_定位与品牌`：统一定位结论、行业话术、产品架构、具体产品、Agent 岗位与 KPI 设计、广告语、`AI Sprint` 方法和销售话术模块
- `04_附件资料`：会议纪要、原始资料、参考案例

## 建议推进顺序

1. 先以 `03_定位与品牌` 里的统一表述作为当前对外口径。
2. 再在 `02_外部研究` 里持续补各行业客户的语言偏好和真实反馈。
3. 如果定位发生变化，再回写 `01_现状盘点` 和本页。

## 文件命名建议

- 过程文档优先使用 Markdown。
- 外部原始文件统一放入 `04_附件资料`。
- 文件名尽量遵循 `主题_YYYYMMDD.md`。

- 对 `AI Agent绩效` 做去粗取精更新，增加 `Agent KPI OS` 与 `Human-controlled Agent KPI OS` 的区分。

### v0.4.0 - 2026-04-10 14:26

- 新增 `行业观察与竞对分析` 板块，开始系统区分基础模型公司、通用智能体公司、智能体中间层公司和专用智能体设计公司。
- 明确 LinearVision 不应把自己放进“大模型研发公司”或“泛智能体平台公司”的框里。
- 增加 2026 年 Q1 赛道观察，强调从聊天助手走向任务闭环、从通用能力走向垂直岗位、从单点应用走向治理与连接层。
- 新增一版初筛名单，并标注哪些公司值得重点关注、哪些只是观察对象、哪些不应作为核心可比对象。

### v0.3.0 - 2026-04-10 14:19

- 新增 `Agent 岗位与 KPI 设计` 方法文档，明确这不是“人的 KPI”或“编排平台选型”，而是把 Agent 当成虚拟岗位来定义职责、边界和绩效。
- 明确 `AI Agent绩效` 的核心，不是看 Agent 多聪明，而是看它是否稳定创造业务结果。
- 将 Agent KPI 收敛为四类核心维度：效率、质量、财务、风险/安全。
- 补充“这是真需求还是伪需求”的判断标准，避免过早为简单自动化任务过度岗位化。
- 保留本轮讨论的参考引用链接，便于后续回看原始讨论。

### v0.2.0 - 2026-04-10 14:13

- 正式增加两条具体产品线：`AI Agent绩效` 与 `AI安全`。
- 开始把“方法论公司”进一步收敛为“有明确产品抓手的应用方案商”。
- `AI Agent绩效` 对应企业在 Agent 扩张后对绩效、可观测性、责任归属和 ROI 的管理需求。
- `AI安全` 对应企业在权限、审计、合规、动作验证和风险控制上的治理需求。

### v0.1.0 - 2026-04-10 13:00

- 可以向 Palantir 学的是结构和方法，不是照搬它的语气和词汇。
- `See / Think / Act` 这套结构本身没有问题，问题在于后面必须逐步补上真实能力、案例和销售机制。
- 当前阶段保留 `With AI` 是对的，因为 LinearVision 还需要让陌生客户在几秒内知道我们属于什么领域。
- 中短期要避免走成“卖人头的 AI 外包”，正确路径是用服务切入，用产品化能力沉淀。
- 后续定位优化重点不再是继续打磨 tagline，而是把这句 tagline 变成产品骨架、案例故事和获客方式。
- `See / Think / Act` 现在已经明确升级为一套可交付的架构框架，而不只是品牌口号。
- `AI Sprint` 已经明确为一套建议中的销售与验证机制，用来替代抽象的“卖架构设计”。
- 当前阶段真正要打败的，往往不是外部竞品，而是客户内部“先凑合搭几个 Agent 跑起来”的选择。
- 当前更现实的销售切入点，不是教育所有企业做事前架构，而是优先服务那些已经进入混乱期、危机期、重构期的客户。

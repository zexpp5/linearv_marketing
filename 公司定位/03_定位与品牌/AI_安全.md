# AI安全

`AI安全` 是 LinearVision 当前第一批具体产品之一。

它面向的是企业在使用多个 AI Agent 后，对权限、审计、合规、动作控制和风险治理的真实需求。

## 产品定位

这是一个面向企业 AI Agent 的安全与治理产品。

它帮助客户回答五个问题：

- 哪些 Agent 能看到哪些数据
- 哪些 Agent 可以执行哪些动作
- 哪些高风险决策必须经过人工审批
- 出了问题之后能不能追溯
- 面对监管、审计和客户追责时，有没有证据链

## 为什么现在需要这个产品

企业早期自己搭 Agent 时，通常先追求“能跑”。

但随着 Agent 增长，以下问题会越来越集中出现：

- 权限给太宽
- 敏感数据访问不清楚
- 高风险动作没有审批
- 不知道哪个 Agent 做了什么
- 出了事故后没有审计线索

`AI安全` 的作用，就是让企业从“凑合用”走向“可控、可审计、可放心扩张”。

## 核心价值

- 让 Agent 的数据访问和动作执行边界明确下来
- 降低权限越界、误操作、信息泄露和合规风险
- 给技术、法务、内控和管理层一个共同可理解的治理框架
- 把 AI 风险从抽象担忧变成可检查、可记录、可整改的问题

## 核心能力方向

- 权限矩阵：定义不同 Agent 可以访问的数据、系统和动作范围
- 敏感动作控制：对转账、审批、报价、合同修改、客户承诺等高风险动作设置人工确认
- 审计追踪：记录 Agent 看了什么、用了什么规则、输出了什么、执行了什么动作
- 风险分级：按数据敏感度、动作不可逆性、业务影响度划分风险等级
- 回滚与处置：定义出错时的阻断、回滚、人工接管和事故处理流程

## 典型适用场景

- 银行、金融、医疗、法务等强监管行业
- 企业已经开始担心合规、权限和责任问题
- 某些 Agent 已经接入核心系统或具备执行动作能力
- 管理层开始问“出了事谁负责”

## 对应的客户阶段

这个产品最适合：

- 混乱期客户
- 危机期客户
- 重构期客户

尤其适合那些已经出现过权限、合规或责任边界问题的企业。

## 与 `See / Think / Act` 的关系

- `See`：识别敏感数据、关键系统、高风险动作和权限边界
- `Think`：判断哪些场景可自动化，哪些必须审批，哪些必须留痕
- `Act`：执行阻断、审批、审计、回滚和事故处置

## 对外怎么讲

### 简短版

`AI安全` 让你的 Agent 不只是能用，而且能被放心地用。

### 更业务版

当 Agent 开始接触真实业务、真实客户和真实动作时，最大的风险不再是“它会不会做”，而是“它做错了谁负责”。`AI安全` 解决的是这个问题。

### 更偏管理层版

如果你的 Agent 已经开始影响客户、财务、审批和合规，下一步不该只加更多 Agent，而应该先把权限、审计和责任边界立住。

## 当前产品边界

这个产品当前不等于一个完整的企业安全平台。

它更适合作为：

- Agent 治理体系的安全底座
- 高风险业务流程的第一层防护
- `AI Sprint` 之后进入正式项目的核心模块之一

## 当前最重要的 MVP

首版优先聚焦：

- 权限矩阵
- 敏感动作审批
- 基础审计日志
- 高风险动作回滚与人工接管规则

# 行业观察与竞对分析

这份文档用于沉淀 `2026 年 Q1` 关于 AI 软件公司、智能体公司和相关初创公司的阶段性观察。

这里的目的不是堆一个大名单，而是先把公司分类框架拉清楚，再判断哪些公司值得作为 LinearVision 的重点观察对象。

## 使用说明

- 本文以当前讨论内容为基础，适合作为初筛名单和内部判断依据。
- 名单中的部分公司适合后续继续做尽调，不应直接当作最终结论。
- 重点不是“谁最火”，而是“谁和我们是同一类公司”。

## 先把公司分清楚

### 1. 基础模型研发公司

这类公司训练底座模型，卖的是模型能力本身。

不属于 LinearVision 的核心可比对象。

### 2. 通用智能体 / 通用任务 Agent 公司

这类公司强调“什么都能做”的研究型或执行型 Agent，通常面向通用场景、多步骤任务或个人 productivity。

它们值得观察，但不是最核心的直接可比对象。

### 3. 智能体中间层 / 编排层 / 连接层公司

这类公司不训练模型，而是把不同模型、工具和业务系统接起来，提供编排、路由、工具调用、监控、权限和集成能力。

这类公司和我们有相邻关系，既可能合作，也可能在部分方案层面竞争。

### 4. 专用智能体设计公司

这类公司面向特定目标、特定职责、特定流程和特定岗位设计智能体，不卖泛能力，而卖明确结果。

这一类才是更值得重点对标的对象。

## 2026 年 Q1 的几个观察

### 1. 从聊天助手转向任务闭环

赛道重心正在从“会聊天、会回答”转向“能不能把一件事真正做完”。

### 2. 从通用能力转向垂直岗位

越来越多公司不再讲万能 Agent，而开始讲：

- 销售 Agent
- 财务 Agent
- 法务 Agent
- 运维 Agent
- 数据分析 Agent

### 3. 平台化和治理能力增强

越来越多公司不只是做一个应用，而是开始做：

- 编排
- 权限
- 观测
- 审计
- 多模型接入

### 4. 模型无关成为一条明确路线

一批公司不训练底座模型，而是在智能体的各个环节设计端点和连接层，使其能适配不同模型厂商。

这条路线对 LinearVision 尤其重要，因为它更接近“把模型变成真实工作系统”的能力。

## 对 LinearVision 的分类判断

LinearVision 不应被放进下面两类：

- 基础模型研发公司
- 泛智能体 / 通用 Agent 公司

LinearVision 更接近这两类交叉的位置：

- 专用智能体设计公司
- 智能体中间层 / 连接层能力提供方

更准确地说：

LinearVision 应该被定义为：

`面向特定岗位、特定职责和特定工作流，设计并交付可治理智能体系统的公司。`

## 筛选标准

以后判断一家公司是否值得作为重点对标对象，优先看这五个问题：

1. 它的智能体是否对应一个明确岗位或职责。
2. 它是否有明确输入、输出和验收标准。
3. 它是否嵌入了真实业务流程，而不是只做问答层。
4. 它是否能量化节省人力、时间、错误率或风险。
5. 它是否依赖行业知识和流程设计，而不是只依赖通用对话能力。

## 哪些公司不应作为核心竞品

以下几类公司不应该和 LinearVision 混在一个比较表里：

- 只做底座模型训练的公司
- 只卖 API、推理服务、算力的公司
- 只做通用聊天助手的公司
- 只提供平台能力，但没有岗位职责落地的公司

## 对 LinearVision 的启发

### 1. 不要把自己说成模型公司

客户最终不会因为你“接了哪个模型”而付钱，而会因为你交付了一个有明确职责和结果的智能体系统而付钱。

### 2. 不要把自己说成泛平台

平台会让定位变宽，但也会让客户听不懂你到底解决什么问题。

### 3. 更适合的路径是“专用智能体 + 连接层能力”

这意味着：

- 对外卖的是明确岗位和结果
- 对内沉淀的是多模型、多工具、多系统的连接能力

### 4. 竞对分析应该服务于产品收敛

看竞对不是为了抄功能，而是为了持续回答两个问题：

- 我们到底不做什么
- 我们最值得先做的专用智能体到底是哪几个

## 后续建议

- 把名单做成“重点跟踪 / 延伸观察 / 排除”三层
